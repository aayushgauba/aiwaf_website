FAQ

Frequently asked questions about AIWAF installation, configuration, and troubleshooting.

How do I install AIWAF?
AIWAF can be installed using our automated installation script. Simply run:
curl -fsSL https://get.aiwaf.com/install.sh | bash
This will download and install all necessary components automatically.
What are the system requirements?
AIWAF requires:
  • Linux-based OS (Ubuntu 20.04+ or CentOS 8+)
  • Minimum 8GB RAM (16GB recommended)
  • 4+ CPU cores with AVX2 support
  • 500GB available disk space
  • Network connectivity for updates
How do I configure AIWAF for my application?
Configuration is done through the YAML file at /etc/aiwaf/config.yml. Start with the default settings and customize the upstream servers, protection rules, and sensitivity levels as needed for your environment.
Can I run AIWAF in Docker?
Yes! AIWAF provides official Docker images. You can deploy using Docker Compose or Kubernetes. Check our Getting Started guide for detailed container deployment instructions.
How do I troubleshoot connection issues?
First, check the service status and logs:
sudo systemctl status aiwaf sudo journalctl -u aiwaf -f
Ensure your firewall allows traffic on the required ports (80, 443, 8080) and that your upstream servers are accessible.
How often are threat intelligence updates delivered?
AIWAF receives threat intelligence updates in real-time. The AI models are updated continuously, and signature-based rules are updated multiple times per day to ensure protection against the latest threats.
